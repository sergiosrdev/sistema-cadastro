# Começando com o Create React App

Este projeto foi criado com [Create React App](https://github.com/facebook/create-react-app).

## Scripts disponíveis

No diretório do projeto, você pode executar:

### `npm start`

Executa o app em modo de desenvolvimento.  
Abra [http://localhost:3000](http://localhost:3000) para visualizá-lo no navegador. [web:4]

A página será recarregada automaticamente quando você fizer alterações.  
Você também verá quaisquer erros de lint no console. [web:4]

### `npm test`

Executa o test runner no modo interativo de observação (*watch mode*).  
Veja a seção sobre [execução de testes](https://facebook.github.io/create-react-app/docs/running-tests) para mais informações. [web:1]

### `npm run build`

Gera o build de produção do app na pasta `build`.  
Ele empacota corretamente o React em modo de produção e otimiza o build para o melhor desempenho. [web:1][web:4]

O build é minificado e os nomes dos arquivos incluem *hashes*.  
Seu app está pronto para ser implantado! [web:1][web:7]

Veja a seção sobre [implantação](https://facebook.github.io/create-react-app/docs/deployment) para mais informações. [web:9]

### `npm run eject`

**Atenção: esta é uma operação irreversível. Depois que você executar `eject`, não há como voltar atrás!** [web:5]

Se você não estiver satisfeito com as ferramentas de build e as configurações fornecidas, pode executar `eject` a qualquer momento.  
Este comando removerá a dependência única de build do seu projeto. [web:3][web:11]

Em vez disso, ele copiará todos os arquivos de configuração e dependências transitivas (webpack, Babel, ESLint, etc.) diretamente para o seu projeto, dando a você controle total sobre eles.  
Todos os comandos, exceto `eject`, continuarão funcionando, mas passarão a apontar para os scripts copiados, permitindo que você os ajuste. A partir desse ponto, a responsabilidade passa a ser sua. [web:3][web:5]

Você não é obrigado a usar `eject`.  
O conjunto de recursos padrão é adequado para pequenos e médios projetos, e você não deve se sentir na obrigação de usar esse recurso. No entanto, entendemos que esta ferramenta não seria tão útil se você não pudesse personalizá-la quando estiver pronto para isso. [web:11]
